========================================================================
                      PORTFOLIO HUB - PROJECT BREAKDOWN
========================================================================

1. OVERVIEW
-----------
The Portfolio Hub is a centralized, high-performance web ecosystem designed 
to showcase 18 specialized technical portraits. It functions as a "Technical 
Universe" where each portal represents a different language, tool, or domain.

2. CORE TECHNOLOGIES USED
-------------------------
- HTML5 (Semantic): Used for high-integrity structural architecture, 
  accessibility (ARIA), and SEO.
- CSS3 (Modern): Employed for glassmorphism effects, advanced layouts 
  (Flexbox/Grid), and smooth micro-animations.
- Tailwind CSS: Used for rapid, mobile-first utility styling and maintaining 
  a consistent design system across 19 pages.
- JavaScript (ES6+): The logic engine for the dynamic DOM, theme management, 
  and content injection.
- Three.js: A high-performance WebGL library used to architect the theme-
  synchronized 3D backgrounds and interactive avatars.
- Vercel/Netlify: Platforms used for global edge deployment and static hosting.
- Google Gemini AI: Integrated via serverless functions for the ATS Resume 
  Score project and the Hub's intelligent chatbot.

3. PROJECT ARCHITECTURE
-----------------------
The project follows a "Hub-and-Pod" architecture:
- [index.html]: The central navigation hub connecting all portals.
- [domain.html]: 18 specialized sub-portfolios (e.g., python.html, cpp.html, 
  figma.html).
- [assets/]: Centralized storage for SVGs, logos, and visual visualizations.
- [functions/]: (Deployment Level) Contains gemini-proxy.js for secure 
  AI API communication.

4. CORE LOGIC & WORKING
-----------------------
A. 3D Background Engine (Three.js):
   - Logic: Initializes a WebGL scene with randomized wireframe geometries 
     (Tetrahedrons, TorusKnots, Boxes).
   - Interaction: Geometries float in 3D space with randomized speeds and 
     rotate/move in response to mouse coordinates.
   - Theme Sync: The background color dynamically swaps between hex values 
     based on the active Dark/Light mode state.

B. The Sentiment Avatar (index.html):
   - Logic: A complex Three.js group consisting of a wireframe Icosahedron 
     shell and a glowing inner core.
   - Intelligence: "Digital Eyes" utilize trigonometric constraints to track 
     the user's cursor across the viewport.
   - Animation: Floating "Satellite Hands" move in a cosine-wave pattern to 
     simulate a living, responsive entity.

C. Theme Management System:
   - Logic: Uses LocalStorage to persist user preference. On toggle, it 
     injects the 'dark' class into the root HTML element, triggering 
     Tailwind's dark: variants and updating Three.js material colors.

D. Dynamic Portal Population:
   - Logic: Instead of hard-coding 100+ project cards, each portal contains 
     a JavaScript array of objects.
   - Execution: A populate() function maps these arrays into HTML templates, 
     allowing for rapid updates and high maintainability.

E. AI Chatbot Integration (Backend Logic):
   - Logic: Uses a secure proxy (Netlify Function) to communicate with the 
     Google Gemini API.
   - Working: Sanitizes user inputs, adds professional system prompts, 
     manages API keys as environment variables, and provides a "Human-AI" 
     bridge for technical inquiries.

5. WHY THESE TOOLS WERE USED
----------------------------
- Vanilla JS over Frameworks: Used to ensure maximum performance, zero build 
  overhead, and direct control over the WebGL rendering pipeline.
- Three.js: To create a "Wow" factor that distinguishes the hub from 
  standard flat portfolios.
- Tailwind CDN: Optimized for static hosting where speed and ease of 
  deployment are paramount.
- Glassmorphism: To give the project a premium, modern aesthetic that aligns 
  with high-end software products.
